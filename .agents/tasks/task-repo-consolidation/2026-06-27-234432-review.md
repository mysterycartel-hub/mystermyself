# Org repo consolidation into mystermyself monorepo

Three satellite repos from mysterycartel-hub collapsed into the main Next.js production repo: TCU terminal legacy docs archived under `docs/archive/`, the Remotion video engine parked in `tools/trading-chef-studio/` with its own package.json, and Pine Script + AI prompts placed in `scripts/trading-chef/`. Build isolation is enforced at three layers (tsconfig exclude, webpack rule, eslint ignorePatterns). The existing Next.js build and lint pass unchanged.

**Watch for:** The webpack exclude rule in `next.config.js` appends a bare `test`+`exclude` rule without a loader, which is a no-op that won't actually prevent Next.js from resolving imports into `tools/` if anything ever references it (confirmed). The real protection comes from tsconfig exclude, but a stray import would produce a confusing webpack error rather than a clean "module not found". Separately, the archived MASTER_PROMPT.md contains a personal email address committed to the repo (confirmed).

## High-level view

The consolidation is strictly additive — 54 new files, one config edit, two new config files, no existing source touched. Build passes because nothing imports the new files; the exclusion rules are a safety net for the future.

The three-layer exclusion strategy (TypeScript, webpack, ESLint) sounds thorough but the webpack layer is misconfigured. It pushes a rule with `test` and `exclude` but no `loader` or `use`, meaning webpack will match the files and do nothing special — it doesn't prevent resolution. The tsconfig exclude is the real gate, but the webpack rule gives false confidence.

The archived docs carry the original repo's assumptions wholesale, including the CEO's personal email in `MASTER_PROMPT.md`. Worth a conscious decision on whether that stays in git history.

The Remotion subdirectory pins all dependencies to `"latest"`, which means any future `npm install` inside that directory will produce a non-reproducible build. This makes the Remotion tool fragile for future use.

## Webpack exclude rule doesn't do what it looks like

The `next.config.js` change pushes this into `module.rules`:

```js
{
  test: /\.(ts|tsx|js|jsx)$/,
  exclude: [path.resolve(__dirname, 'tools')],
}
```

A webpack rule with `test` and `exclude` but no `loader` or `use` property is effectively a no-op. Webpack matches the file against existing loaders in order; this rule doesn't install a new loader or suppress an existing one. If a file inside `tools/` were ever imported (directly or transitively), webpack would still process it using Next.js's built-in SWC loader because that loader's own `include`/`exclude` config is what actually gates compilation.

The tsconfig `"exclude": ["node_modules", "tools"]` is the real gate — TypeScript won't type-check `tools/`, so Next.js won't compile it. The webpack rule creates a false sense of defense-in-depth.

To actually block webpack resolution into `tools/`, use a `resolve.alias` pointing `tools` to `false`, or a null loader (`use: 'null-loader'`), or drop the webpack rule entirely and rely on tsconfig exclude plus the discipline of never importing from `tools/`.

## Remotion dependency pinning

Both `tools/trading-chef-studio/package.json` and `tools/trading-chef-studio/trading-chef-remotion-starter/package.json` specify `"latest"` for every dependency:

```json
"dependencies": {
  "@remotion/cli": "latest",
  "@remotion/renderer": "latest",
  "remotion": "latest",
  "react": "latest",
  "react-dom": "latest"
}
```

The README says to run `npx remotion studio` and the package.json has render scripts. First use will install whatever `latest` means that day, and Remotion has breaking changes between major versions. Pinning to known-good versions (or at minimum a major range like `^4.0.0`) would make this tool reproducible.

## New `.eslintrc.json` introduces a project-wide rule change

No `.eslintrc.json` existed on `main`. The new file was created to add `ignorePatterns: ["tools/"]`, which is the stated purpose. But it also adds `"react/no-unescaped-entities": "off"` as a project-wide rule, disabling a lint rule that catches unescaped quotes in JSX across the entire codebase. This is an unrelated change bundled into a consolidation PR — it likely masks pre-existing warnings in the app code rather than being necessary for the tools exclusion.
