## FIX APPLIED

**File:** `app/opportunity-list/page.tsx`

**Change:**
1. Added `import { Suspense } from 'react'` at top of file
2. Wrapped `<OpportunitySignup source="opportunity_list_page" />` in `<Suspense fallback={null}>`

**Before:**
```tsx
<OpportunitySignup source="opportunity_list_page" />
```

**After:**
```tsx
<Suspense fallback={null}>
  <OpportunitySignup source="opportunity_list_page" />
</Suspense>
```

**Why:** Next.js App Router requires any component using `useSearchParams()` to be rendered inside a `<Suspense>` boundary. The `OpportunitySignup` component uses `useSearchParams()` to pre-fill the lane dropdown from URL params (`?lane=interest_trading_chef`). Without Suspense, Next.js throws a build error because it can't determine the search params at static build time.
